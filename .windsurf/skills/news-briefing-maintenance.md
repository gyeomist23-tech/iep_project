# Skill: News Briefing Maintenance

## Purpose
이 스킬은 이 저장소에서 반복적으로 수행되는 유지보수 작업을 빠르게 처리하기 위한 체크리스트다.

## When to Use
- 키워드 세트를 추가 또는 조정할 때
- 기사 분류가 기대와 다르게 나올 때
- 너무 오래된 기사가 출력될 때
- 지역 교육청 기사 비중이 높을 때
- 출력 포맷이 깨졌을 때
- GitHub Actions 자동 커밋이 동작하지 않을 때
- 사용자가 새 섹션이나 제한 수 변경을 요청할 때

## Procedure
1. 변경 목적을 `수집`, `필터링`, `분류`, `출력`, `자동화` 중 하나로 식별한다.
2. 관련 파일만 최소 범위로 수정한다.
3. 아래 항목을 기준으로 회귀 여부를 점검한다.
   - 링크 전역 dedupe 유지
   - 최근 기사 기간 필터 유지
   - 선호 출처 우선 반영
   - 지역 교육청 계열 출처 허용 여부와 우선순위 유지
   - 섹션 우선순위 유지
   - 섹션 순서 유지
   - 빈 결과 파일 생성 가능
   - HTML 출력과 루트 `index.html` 동시 생성
   - README 실행 예시 최신 상태 유지
4. 필요 시 로컬 실행 후 생성된 텍스트 파일을 직접 열어 사람이 읽는 포맷을 검토한다.

## Common Fix Patterns
### 1. 새 키워드 추가
- 해당 섹션의 키워드 배열에만 추가한다.
- 제목 필터 대상이어야 하면 `MUST_HAVE_TERMS` 반영 여부도 검토한다.

### 2. 분류 충돌 수정
- 우선순위 로직을 먼저 검토한다.
- 섹션 키워드 중 중의적인 단어가 있는지 확인한다.

### 3. 오래된 기사 노출 수정
- RSS entry의 `published` 또는 `updated` 필드를 우선 확인한다.
- 실행 시점 기준 허용 기간 기본값이 28일인지 확인한다.
- 날짜를 해석할 수 없는 기사는 기본적으로 제외되는지 확인한다.

### 4. 출처 품질 수정
- 기사 제목 또는 RSS source 필드에서 매체명을 추출하는지 확인한다.
- 교육 전문지, 메이저 언론, 학술지 계열이 우선 반영되는지 확인한다.
- 지역 교육청, 교육지원청, 교육지원센터 계열은 허용하되 우선순위가 낮은지 확인한다.

### 5. Actions 커밋 문제 수정
- `git add output` 범위 확인
- `git diff --cached --quiet` 조건 확인
- `permissions.contents: write` 유지 여부 확인

### 6. 출력 포맷 오류 수정
- 줄바꿈 수와 섹션 사이 빈 줄 개수를 먼저 확인한다.
- 사용자 요구 포맷과 문자열이 한 글자라도 다른지 비교한다.
- TXT와 HTML의 기사 목록이 서로 다르지 않은지 함께 확인한다.

## Definition of Done
- 사용자가 요구한 포맷과 동작이 유지된다.
- 불필요한 기능이 추가되지 않는다.
- 자동화와 문서가 서로 일치한다.
